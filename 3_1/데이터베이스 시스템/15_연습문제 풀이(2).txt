 # 연습문제 풀이.

물리적 저장장치 데이터 접근 속도. ( 빠른순서 )
레지스터 -> 캐시 -> 메인메모리 -> 자기디스크.

힙 파일구조.
모든 레코드를 파일 내 임의의 위치에 저장하며, 저장하는 순서를 고려하지 않는 파일 구조.

슬롯 페이지 구조.
가변 길이 레코드를 저장하기 위해 파일 또는 블록의 헤더의 헤더에 유지하는 형식의 구조.

인덱스의 효율성에 대한 평가기준.
새로운 데이터 삽입 시 발생하는 인덱스 구조 유지 비용.
인덱스를 통해 데이터를 찾고 접근하는데 걸리는 시간.
인덱스를 저장하기 위해 부가적으로 필요한 공간 비용.
오답 : 사용자의 인덱스를 사용하는 질의 요청 빈도. ( 상관 없음 )

밀집 인덱스.
모든 탐색키 값에 대해 탐색키 <값,포인터>쌍으로 구성된 인덱스 엔트리를 갖는 인덱스로 
인덱스 파일의 크기가 커서 I/O 비용이 증가하여 탐색 시간이 오래 걸릴 수 있는 단점을 지님.

해시 함수.
검색키를 버킷 주소에 대응시키는 함수.

동거자.
서로 다른 두 레코드 R1과 R2의 검색키가 h에 의해 동일한 버킷으로 대응되었을 때를 의미.

동적 해싱.
데이터베이스의 크기에 따라 버킷의 개수가 조절되는 형태의 해싱.

비트맵 인덱스.
컬럼에 대해 값을 0과 1로 표현하는 것.

트랜잭션의 특성 ACID.
원자성, 일관성, 독립성( 고립성 ), 지속성.
오답 : 확장성.

트랜잭션을 동시에 실행시키는 목적.
1. 데이터의 가용성을 향상시킬 수 있다.
2. 트랜잭션의 대기시간을 감소시킬 수 있다.
3. 트랜잭션의 처리율과 자원의 이용률이 향상된다.
오답 : 데이터베이스의 일관성이 향상된다.

사이클.
스케쥴에 대기 그래프에 사이클이 존재하면 교착상태에 있다는 것을 의미한다.

희생자.
교착상태를 해결하기 위해 롤백시킬 트랜잭션을 의미한다.

Write Ahead Log.
Write 연산을 수행할 때 마다 데이터베이스가 변경되기 전에 로그 레코드를 로그에 추가하는 방식.

Redo를 수행해야 하는 트랜잭션.
checkpoint 이후에 commit, abort 로그가 있는 트랜잭션.
