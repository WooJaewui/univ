
 HTML 개요.



 # HTML이란 ?

HTML HyperText Markup Language.
WEB World Wide Web에서 사용하는 마크업 언어.
인터넷 상에서 사용되는 웹페이지의 정보를 표현하기 위한 언어.

하이퍼텍스트 HyperText.
다른 텍스트에 대한 참조( "링크" )를 가진 텍스트.
단위 텍스트들과 이것들을 결합하는 링크로 구성.
사용자가 연상하는 순서에 따라 관련 있는 쪽으로 이동하여 정보를 얻도록 조직화된 시스템.

마크업 MarkUp
태그를 의미한다.
문서의 구조와 내용 들에 추가적인 정보를 부여하기 위해 문서 내에 삽입되는 일련의 문자 또는 기호.

문서의 구성.
내용 content : 문서의 실제 본문 내용.
구조 structure : 문서의 구성 정보.
스타일 style : 문서의 표현.

마크업의 분류.
의미적 마크업 semantic. : <title>, <address>, ...
구조적 마크업 structural : <p>, <div>, ...
양식적 마크업 stylistic : <b>, <i>, <u>, ...

마크업 언어.
마크업의 형식과 규칙을 정의한 언어.

마크업 언어의 분류.
절차적 마크업 언어 : 문서의 형식이나 텍스트의 외양을 기술. ( 워드프로세서에서 사용 - 호환성 문제 )
기술적 마크업 언어 : 문서의 논리적인 구조 정보 및 속성을 표현. ( SGML, HTML, XML 등 - 범용적이다 )

SGML Standard Generalized Markup Language.
이기종 장치 및 시스템에 독립적으로 전자 텍스트를 표현할 수 있는 국제 표준 마크업 언어 ( ISO 8879 )
매우 강력한 표현력 -> 문법과 구성이 너무 복잡.
언어의 습득과 응용 시스템 구현의 어려움.
군사, 우주, 항공 등에 제한적으로 사용.

XML eXtensible Markup Language.
SGML로부터 필요한 기능만 뽑아서 새롭게 정의한 언어.
사용자에 의한 임의의 태그 정의가 가능. ( 사용 목적과 용도에 맞는 문서 작성이 가능 )
HTML과 달리 문서의 논리적 구조를 정의하는 언어.
문서 내용의 용이한 이해 및 명확한 전달 가능.

HTML 장점.
수정 / 관리 용이, 웹 접근성 향상, 브라우저 등의 호환성 확보.
파일 용량이 작아 클라이언트-서버 간의 빠른 문서 전달 가능.
별도의 전용 프로그램이 아닌 간단한 편집기로도 작성 가능.
텍스트 문서 -> 다양한 시스템 / 브라우저에서도 사용 가능.

HTML 단점.
웹 문서의 내용 표현에 집중.
정보의 구조화 및 데이터 간의 연관성 표현의 어려움.
사용 가능한 태그가 정해져 있음.
문서의 유효성 검증과 제약조건 정의가 어려움.

W3C
웹의 장기적인 성장을 보장하기 위한 공개 표준을 개발하는 국제적인 커뮤니티.

WHATWG.
웹을 통해 애플리케이션을 작성하고 전개할 수 있는 새로운 기술 개발을 위해 만든 공개 커뮤니티.

W3C 권고안 추진 단계.
노트 Note : 기술 문서로 제안하기 위해 제시된 문서.
초안 Working Draft : 아직 완전하기 않은 WG의 아이디어를 담은 문서.
후보 권고안 Candidate Recommendation : 심사를 거친 작업 최종안.
제안 권고안 Proposed Recommendation : 최종적인 권고안이 될 수 있는 전 단계.
권고안 Recommendation : W3C에 참여하는 회원에게 동의를 얻은 표준안.

HTML 서비스의 구성 및 처리 과정.
URL 입력 -> DNS서버 -> IP 주소 변환 -> HTML 페이지 요청 -> 웹 서버 
-> 페이지 요청 -> 페이지 전송 -> HTML 파일 전송 -> HTML 태그 분석 및 변환.

 # HTML 문서 작성의 기초.

HTML 문서의 구성 요소.
하나의 HTML 문서는 다양한 요소( element )들의 조합으로 구성.
요소( element )는 시작태그, 속성, 내용, 종료태그 등으로 구성되어 있다.
종료태그 없이 사용하는 element도 있다.

HTML 문서 작성 시 주의사항.
태그는 대소문자를 구분하지 않음. ( 소문자 표기를 권고 )
파일의 확장자는 반드시 htm 또는 html로 지정. ( 파일명.htm, 파일명.html )
화이트스페이스는 입력한 의도대로 적용되지 않음. ( 두 칸 이상의 연속된 공백은 하나의 공백으로 처리 )
요소 안에 다른 요소를 포함할 수 있음.

문서유형 DOCTYPE 선언.
브라우저가 웹 페이지를 올바르게 표시할 수 있도록 문서 형식을 알려주는 지시어.

<html>
브라우저에게 HTML 무넛의 시작과 끝을 알려주기 위한 것.

<head>
문서의 각종 정보와 문서 자체에 대한 설명을 포함. ( 화면에 포함되지 않음 )
<title>, <style>, <link>, <script>, ... 등을 사용.

<body>
브라우저 화면에 실제로 출력해서 보여주려는 모든 내용을 포함.

<title>
웹 브라우저 상단에 HTML 문서의 제목을 표시할 때 있다.

<link>
별도 파일로 작성된 CSS 스타일을 웹 문서에 적용할 때 사용.

<meta>
웹 문서의 문자 인코딩 방식, 문서에 대한 간략한 설명 등의 다양한 정보를 표현.

<base>
웹 문서의 기본적인 URL 경로 지정.
<base target="_blank"> <a href="abc.html"> 등으로 활용할 수 있다. ( 새로운 창을 띄움 )
