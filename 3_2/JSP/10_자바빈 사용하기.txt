
 자바빈 사용하기.



 # 자바빈 설계 규약.

자바빈 정의.
JSP 요소들과 맞물려 사용 가능한 사용자 정의 자바 클래스.
자바빈 설계 규약에 따라 작성해야 함.

사용 이유.
JSP 페이지에서 사용할 데이터를 다루려는 목적.
비즈니스 로직과 프레젠테이션 로직의 분리.
HTML과 JSP 기본 문법으로 프레젠테이션 로직.
자바빈을 이용하여 비즈니스 로직을 구현.
여러 JSP 페이지에서 자바빈을 재사용.
자바빈은 별도의 클래스 파일로 존재함.

자바빈 설계 규약.
생성자가 필요하면 파라미터가 없는 기본 생성자를 반드시 만들어야 함.
읽기와 쓰기가 가능한 속성을 가질 수 있음.
모든 속성에 대해 getter 메소드와 setter 메소드를 정의.
getter 메소드는 파라미터가 없으며 속성의 타입과 일치하는 리턴 타입을 가짐.
setter 메소드는 속성의 타입과 일치하는 하나의 파라미터를 가지며 리턴 타입은 void.



 # 자바빈 사용.

자바빈 클래스 만들기.
New -> Class -> 패키지, 클래스 이름을 입력하고 Finish.
자바빈 클래스를 작성.

이클립스에서 파일의 위치.
.class 파일은 [workspace]\build\classes\ 폴더에 위치함.
.java 파일은 [workspace]\src\main\java\ 폴더에 위치함.

자바빈 관련 액션 태그.
<jsp:useBean> : 자바빈 객체의 생성 또는 사용하겠다는 선언.
<jsp:setProperty> : 자바빈 객체의 속성을 설정함.
<jsp:getProperty> : 자자빈 객체의 속성을 읽음.



 # 액션 태그.

<jsp:useBean>
JSP 페이지에서 자바빈을 사용할 것이라는 선언.
사용 범위에서 빈이 존재하면 빈을 가져다가 사용한다.
사용 범위에서 빈이 존재하지 않으면 빈을 생성하며 해당 scope 객체의 속성으로 저장.
id : JSP 페이지에서 사용될 자바빈 객체 이름.
class : 자바빈의 유형으로 사용되는 클래스의 이름.
scope : 자바빈 객체가 사용되는 범위. ( page, request, session, application 중 하나 )

<jsp:setProperty>
자바빈 객체의 속성 값을 변경.
setter 메소드와 대응됨.
name : 자바빈 이름.
property : 속성 이름.
value : 속성 값.
<jsp:setProperty>가 <jsp:useBean>의 내부에서 사용될 때는 자바빈 객체가 생성되는 경우에만 실행.

<jsp:getProprty>
자바빈 객체의 속성값을 읽음.
getter 메소드와 대응됨.
name : 자바빈 이름.
proprty : 속성 이름.
