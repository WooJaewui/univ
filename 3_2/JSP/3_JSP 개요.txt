
 JSP 개요.



 # JSP 기술.

JSP.
동적으로 웹 컨텐츠를 만들기 위한 기술.
동적으로 웹 페이지를 만드는 기술.
서버 측 스크립트 언어.
HTML 페이지 안에 자바 코드를 포함시킬 수 있음.
서블릿 코드를 작성하는 편린한 방법을 제공.

JSP 언어의 확장 메커니즘.
표현 언어 EL의 사용.
서버 측 객체나 데이터를 쉽게 사용하고 표현할 수 있음.
JSTL과 같은 태그 라이브러리의 사용.

JSP 문서.
템플릿 데이터 : 정적 테이터로 HTML, XML 형식의 텍스트.
JSP 요소 : 적절한 Java 코드로 번역되어 동적 컨텐츠가 만들어짐.



 # JSP 문서의 구성.

JSP 페이지의 구성 요소.
스크립트 요소.
지시어.
액션 태그.
내장 객체.
표현 언어.
JSTL과 사용자 정의 태그.
JSP 주석 <%-- 설명 %>

스크립트 요소.
스크립트릿 : <% 자바코드 %>
표현식 : <%= 출력자바코드 %>
선언 : <%! 변도 또는 메소드 선언 자바코드 %>
지시어 : <%@ 정보 자바 코드 %> ( directive )

지시어.
include : JSP 페이지의 특정 영역에 다른 문서를 포함.
page : JSP 페이지에 대한 정보를 지정.
taglib : JSP 페이지에서 사용할 태그 라이브러리를 선언함.

내장객체.
자주 사용되는 기능을 JSP 컨테이너가 객체 형태로 제공한 것.
별도의 선언이나 초기화 없이 사용 가능.
request, response, session,application, out, page 등.

액션태그.
요청을 처리할 때, 특별한 기능을 수행하는 것.
형식은 XML 태그와 같은 모양.
표준 액션 태그는 접두어로 jsp를 사용.
<jsp:액션이름 속성="값" ...> 몸체 </jsp:액션이름>

표현언어.
데이터를 쉽게 다루기 위한 간단한 스크립트 언어.
형태는 ${expr}이며 값을 표현. ( 표현식보다 사용하기 쉬움 )
스크립트 요소 밖에서 사용. ( 액션 태그 속성 값을 설정, JSTL와 함께 사용 등 )

JSTL과 사용자 정의 태그.
JSTL : 자주 사용되는 기능을 태그로 만듬. ( 조건문, 반복문 등 )
사용자 정의 태그 : 사용자가 정의한 커스텀 태그. ( JSP 표준 태그를 확장하기 위한 것 )



 # page 지시어.

용도.
JSP 페이지에 대한 정보를 지정하여 JSP 처리에 사용하게 함.
사용할 스크립트 언어, 상속받을 클래스, import할 클래스, JSP가 생성하는 문서의 타입 등.

주요 속성.
autoFlush : true일 경우 버퍼가 차면 자동으로 버퍼를 비운다. ( false일 경우 버퍼가 차면 예외 발생 )
contentType : 결과로 생성되는 응답의 종류와 사용될 charset 정보를 설정.
import : JSP 페이지에서 사용할 자바 클래스를 지정.
language : JSP 페이지의 스크립트 요소에서 사용할 프로그래밍 언어를 설정.
session : JSP 페이지에서 session 객체를 사용할 지의 여부를 지정.
buffer : JSP 페이지의 출력 버퍼 크기를 지정. 'none'일 경우 버퍼를 사용하지 않음.
info : JSP 페이지에 대한 설명을 텍스트로 입력할 때 사용.
errorPage : JSP 페이지 실행 중 에러가 발생할 때 보여줄 페이지를 설정.
isErrorPage : 현재 페이지가 에러 페인지인지의 여부를 지정. ( true - 에러페이지 사용 )
pageEncoding : JSP 페이지 자신의 문자 인코딩을 지정.
isELignored : 표현 언어를 사용할 지를 지정.
trimDirectiveWhitespaces : 생성되는 문서에서 불필요할 문자를 제거할 지를 지정.
