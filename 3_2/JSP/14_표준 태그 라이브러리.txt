
 표준 태그 라이브러리.



 # 커스템 태그.

액션 태그.
JSP에서 제공되는 표준 액션.
JSP 페이지의 실행 중에 특별한 액션을 수행함.
XML 문법을 따름.

사용자 정의 태그.
사용자가 정의하는 태그로 커스텀 액션 태그라 함.
회사나 개인이 자신만의 태그 라이브러리를 개발할 수 있음.
태그 라이브러리란 커스텀 태그들의 집합.

JSTL JavaServer Standard Tag Library.
표준화된 커스텀 태그들의 집합.
많은 JSP 응용에서 자주 사용되는 커스텀 태그들을 모은 것.
JSTL에서 정의한 태그를 사용하려면 이것을 구현한 자바 클래스들과 TLD파일을 가지고 있어야 함.

커스텀 태그의 형태.
액션 태그와 커스텀 태그는 모두 XML 태그임.
시작 태그와 종료 태그를 가짐.
속성의 값은 따옴표 사용.
태그 이름의 충돌을 피하기 위해 접두어 prefix를 사용.



 # JSP 표준 태그 라이브러리.

JSTL의 분류.
코어 - c : 변수, 흐름제어, URL 처리.
XML - x : XML 관련 처리.
국제화 - fmt : 메시지 형식, 숫자 및 날짜 형식.
데이터베이스 - SQL : SQL.
함수 - fn : 컬렉션 처리, String 처리.



 # 코어 태그 라이브러리.

JSTL 코어 태그 라이브러리.
set : 사용될 변수를 설정.
remove : 설정된 변수를 제거.
if : 조건에 따라 내부 코드를 수행.
choose : 다중 조건을 처리할 때 사용.
forEach : foreach 구문과 같은 기능을 수행.
forTokens : 구분자로 분리하여 문자열을 처리할 때 사용.
import : URL이 지정하는 자원을 삽입.
redirect : 지정된 URL로 이동.
url : URL을 재작성.
catch : 예외 처리.
out : 출력.

c:set 태그.
<c:set var="변수이름" value="변수의 값" [scope="영역"] />
<c:set var="변수이름" [scope="영역"]>변수의 값</c:set>
<c:set target="${m}" property="name" value="값" />

${변수이름}와 #{변수이름}
${ } : 즉시 값을 계산하여 변수에 저장함.
#{ } : 변수가 사용될 때 값을 계산함. ( 필요할 때마다 값을 계산 )
