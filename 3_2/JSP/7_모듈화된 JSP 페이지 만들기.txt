
 모듈화된 JSP 페이지 만들기.



 # <jsp:include> 액션 태그.

정적 include 방법.
번역 과정에서 파일의 내용인 텍스트 자체를 포함시킬 때 사용.
<%@ include file="파일경로" %>

동적 include 방법.
<jsp:include> 액션태그.

<jsp:include> 액션 태그
지정한 페이지를 태그가 위치한 부분에 포함시키되 해당 페이지의 실행 결과를 포함시킬 때 사용.
JSP 페이지가 실행될 때 처리됨.
including 페이지에서 요청이 보내지고, included 페이지가 실행된 후 결과가 including 페이지 응답에 포함.

JSP 페이지의 모듈화.
공통으로 사용되는 페이지들은 코드의 중복을 피하기 위해 분리할 필요가 있음.
별도 페이지로 만들고 <jsp:include>를 이용.



 # <jsp:param> 액션 태그.

<jsp:param> 태그의 사용.
<jsp:include>나 <jsp:forward>를 사용할 때 요청이 전달됨.
<jsp:param> 태그를 사용하면 요청이 전달될 때 파라미터를 추가 할 수 있음.
<jsp:include>, <jsp:forward>의 서브 엘리먼트로 사용함.

파라미터의 우선순위.
기존 파라미터보다 <jsp:param> 태그에서 정된 파라미터가 우선함.
같은 이름으로 파라미터를 보내고 사용하면 <jsp:param>을 읽는다.



 # include 지시어.

include.
지정된 파일의 소스 코드나 텍스트를 지시어 위치에 삽입시키는 것.
JSP 페이지를 서블릿 프로그램으로 변환시키기 전에 삽입.
JSP 페이지에서 공통으로 사용되는 스크립트 요소를 포함시킬 때 사용.

include 지시어.
include 지시어는 단순히 텍스트를 포함.
include 지시어는 including한 페이지 결과값에 영향을 줄 수 있다.
include 지시어는 공통의 스크립트 요소를 정의할 때 사용.

include 액션태그.
include 액션 태그는 including한 페이지 결과값에 영향을 줄 수 없다.
include 액션 태그는 JSP 페이지를 모듈화하여 구성할 때 사용한다.



 # <jsp:forward> 액션 태그.

<jsp:forward>.
하나의 JSP 페이지에서 다른 페이지로 강제 이동시키는 것.
원래 페이지에서 사용했던 request 기본 객체와 response 기본 객체가 이동 페이지로 넘어감.
원래 페이지의 실행은 포워딩되면서 종료됨. 버퍼의 내용은 지워짐.
원래 페이지를 대신하여 다른 페이지를 실행하는 것.
이동 전과 후의 두 페이지는 같은 request 영역에 존재함.
웹 브라우저의 주소란에는 원래 페이지의 URL이 남아 있음. ( 클라이언트는 이동을 알 수 없다 )

<jsp:forward> 버퍼링 사용.
포워딩 전에 버퍼의 내용을 모두 지움.
포워딩 전에 이미 버퍼가 차서 클라이언트에게 전송된 적이 있다면 예외가 발생함.

<jsp:forward> 버퍼링 사용 X.
포워딩 전에 이미 클라이언트에게 결과를 전송한 적이 있다면 예외가 발생함.
사실상 <jsp:forward>를 사용하려면 버퍼를 써야 함.
