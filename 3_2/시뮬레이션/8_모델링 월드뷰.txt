
 모델링 월드뷰.



 # 모델링 World View.

모델링 월드뷰.
시스템을 모델링하는 관점 framework.

모델링 월드뷰 종류.
이산형, 연속형, 혼합형.



 # 이산형 모델링.

이산형 모델링.
주요 종속변수가 시뮬레이션 시간 구간에서 이산적으로 변한다.
대기행렬 시스템에서 대기 고객수.

모델 예시.
시스템 : 대기행렬.
엔티티 : 고객.
속성 : 도착, 봉사시간.
액티비티 : 봉사활동.

이산형 모델의 월드 뷰.
프로세스 중심 진행방법.
액티비티 중심방법.
사건 중심 진행방법.

프로세스 중심 진행방법.
프로세스를 통한 엔티티의 흐름을 기술.
이해는 쉽지만 모델링의 융통성이 떨어진다.

액티비티 중심방법.
액티비티 시작, 종료조건 규정.
매 시간 관찰, 기간 무한정, 규정된 조건이 만족되어야 액티비티 기간이 결정.
비효율적이다.

사건 중심 진행방법.
사건 발생 때 마다 진행.
시스템의 상태변화.
사건 발생 시간에만 변화.
주요 상태변수 계산.
엔티티와 사건을 표로 만들어서 작성.

전체 대기시간 구하는 공식.
totque = totque + queue * ( time - tlevent )

시간 진행법.
고정 시간 진행법 : 일정한 시간을 정해놓고 진행하는 방법. ( 1초에 한번씩 entity를 확인 )
가변 시간 진행법 : 다음 사건 시간 진행법. ( 사건이 일어난 시간에만 entity를 확인 )



 # 연속형 모델링.

연속형 모델링.
주요 종속변수가 시뮬레이션 시간 구간에서 연속적으로 변한다.
종속변수의 이산적 변화가 어떤 시간에도 발생. ( 예 - 미분방정식 )



 # 혼합형 모델링.

혼합형 모델링.
종속 변수가 시뮬레이션 시간 구간에서 이산적으로도 연속적으로도 변할 수 있다.
이산적인 값의 변화 발생 예 - 발전소 완공 시 순간적으로 에너지 증가, 기상 이변 등.
