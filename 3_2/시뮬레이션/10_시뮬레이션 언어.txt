
 시뮬레이션 언어.



 # 시뮬레이션 언어.

범용언어의 장점.
사용자에게 익숙.
모든 컴퓨터에서 이용이 가능.
적은 수행시간. ( 다양한 분석도구 등이 필요 없는 경우 )
많은 융통성이 허용. ( 복잡한 수치계산 등이 가능 )

범용언어의 단점.
복잡한 대형 시스템 시뮬레이션.
모델링의 어려움.
모델의 타당성 문제.
프로그램의 정확성 문제 검증.

시뮬레이션 언어의 특징.
범용 고급언어보다 더 적합.
모델링을 위한 자연스런 구조와 필요한 모든 기능을 제공.
모델을 에러 없이 프로그래밍으로 변환.
모델링과 프로그래밍 시간 감소.

시뮬레이션 언어의 종류.
GASP.
SLAM, AweSim.
SIMSRIPT.
GPSS.
SIMAN( ARENA )
SIMULA.

시뮬레이션 언어의 선택기준.
컴퓨터에서 언어의 이용가능성.
컴퓨터시스템의 형태/비용.
문서화 정도.
배우기 쉬운 정도.
컴퓨터 수행시간의 효율성.
유연성 및 능력.

시뮬레이션 언어의 요구사항.
모델링 융통성.
모델개발의 용이성.
속도 및 모델의 크기.
컴퓨터와의 독립성.
연속, 이산, 혼합형 모델링.
애니메이션 ( Concurrent Mode 기능이나 Playback Node 기능 )
제공되는 통계적 자료.
사용자 지원.
출력결과 형태의 다양성.

최근의 추이.
PC, workstation, mini computer, mainframe.
애니메이션 가능.
graphic input 가능.



 # AweSim.

AweSim.
Control 부분 : 일반적인 상황기술.
Create 노드 : 도착간격, expon ( 5.0 ), 도착시간기록.
Await 노드 : 대기열, 봉사시간.
Resource 노드 : Server 1번, 대기열번호 1번.
Free 노드 : 사용한 리소스 해제.
Collect 노드 : 데이터 수집, 대기시간 계산.
Terminate 노드 : 종료조건 1,000명.



 # ProModel.

ProModel.
1980년대 Charley Harrell 박사가 개발.
사용하기가 쉽고 특히 그래픽 애니메이션이 뛰어나다.



 # AutoMod.

AutoMod.
1982년 미국 오토 시뮬레이션사에서 개발.
3차원 전용 시뮬레이션 언어.
물류 진단.
공장설비 진단.
자동창고 시스템.
공장자동화.
로보트 시스템.

AutoMod 특징.
Model reality 구현.
3D Graphics 및 Animation.
자유로운 Animation 편집.
GUI 환경제공.
확장된 통계적 기능.
풍부한 Movement System 제공 ( 공장자동화 )
CAD의 도면을 직접 모델링에 사용가능.



 # OPNET.

OPNET Reverbed Modeler.
1986년 미 국방성 프로젝트, MIT에서 개발.
네트워크 시뮬레이션 언어.
네트워크 장치.
네트워크 응용프로그램.
네트워크 서버.
가상 네트워크 환경을 소프트웨어 상에서 구현하여 시뮬레이션 한다.

OPNET의 특징.
광범위한 표준 모델 라이브러리 제공.
GUI 및 자동 토폴로지 구성 기능.
Hybred 시뮬레이션 제공 ( DES, Analytic )
네트워크 구성 상태 및 결과를 웹으로 출력.
시뮬레이션 결과 - 토표, 그래프, 애니메이션.
계층적 네트워크 모델링 ( subnet 모델 )

OPNET 활용분야.
네트워크 전반.
네트워크 상의 문제해결.
어플리케이션의 성능 분석 및 진단.
새로운 통신망 알고리즘이나 프로토콜 검증 및 성능 분석.



 # 애니메이션.

애니메이션의 장점.
경영자/관리자에게 시뮬레이션 설명 용이.
시뮬레이션 프로그램의 에러 수정 용이.
시뮬레이션 모델 검증.
시스템 성능을 위한 개선점 착안.
교육용.
프로젝트 팀원간의 커뮤니케이션 향상.

애니메이션의 문제점.
애니메이션이 모델 정확도를 보장해주지는 못한다.
애니메이션으로 시뮬레이션 결과에 대한 통계적 분석을 대신할 수는 없다.
