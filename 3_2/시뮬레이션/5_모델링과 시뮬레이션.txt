
 모델링과 시뮬레이션.



 # 시스템과 모델.

모델링과 시뮬레이션.
실시스템 -> 모델링 -> 모델 -> 시뮬레이션 -> 시뮬레이터( 컴퓨터, 모델하우스 등 )

시뮬레이션 모델의 이용 범주.
설명적장치 : 시스템이나 문제를 정의.
분석도구 : 한계적 구성요소를 결정.
설계평가도구 : 제안된 해결방안을 종합하고 평가.
예측도구 : 미래의 개발계획을 예측.

시스템 system.
어떤 목적을 위하여 하나 이상 서로 관련된 구성요소들이 결합된 집합.
시스템의 범위 결정 문제. ( 외적 요인들을 어떻게 표현할지 - 포함, 무시, 입력변수 등 )

모델 model.
시스템을 서술한 것.



 # 모델 설계.

모델 설계가 용이한 경우.
물리적 규칙이 이용 가능하다.
도형적 표현이 가능하다.
입력, 출력, 구성요소의 변화가 통제 가능하다.

모델 설계가 어려운 경우.
기복 규칙이 없다.
표현하기 어려운 많은 절차적 요소.
랜덤( random )구성 요소.
정량화가 어려운 정책적인 입력.
인간의 의사결정이 큰 영향을 주는 경우.

모델설계 과정( 모델링 )
모델작성 목적 수립.
시스템 영역 확정. ( 어느정도 범위까지 표현할지 )
모델 상세성 확정. ( 얼마나 자세히 표현할지 )
평가척도 및 재설계. 



 # 시뮬레이션 모델의 종류.

결정적 모델과 확률적 모델.
결정적 모델 : 확률변수가 없어도 구현할 수 있는 모델.
확률적 모델 : 확률변수가 있어야만 구현할 수 있는 모델.

정적 모델과 동적 모델.
정적 모델 : 시간과 무관하다. ( 몬테칼로 시뮬레이션, 주사위 문제 )
동적 모델 : 시간의 흐름에 따라 동적으로 변함.

이산 모델과 연속 모델.
이산 모델 : 값이 띄엄띄엄 있는 모델. ( 대기 행렬 문제 )
연속 모델 : 시간에 상관없이 연속적으로 값을 가지는 모델. ( 공의 탄성 문제 )

물리적 모델과 수리적 모델.
물리적 모델 : 모델하우스, 집 등과 같이 물리적으로 표현할 수 있는 모델. ( 축소모형, 진흙 모델 )
수리적 모델 : 숫자 등과 같이 수리적으로 표현할 수 있는 모델. ( 저축문제, 공의 탄성문제 )



 # 몬테칼로 시뮬레이션 Monte Carlo Simulation.

반지름 1인 원의 면적을 구하기.
전체 모래알 = limit
사각형 면적 = 1
A내 모래알 = count
A의 면적 = 파이/4

전체모레알 : 원내 모레알.
1 : 파이/4 = limit : count.
파이 = count/limit * 4



 # 시뮬레이션.

시뮬레이션의 특징 - 장점
실제 구축 없이 예측 평가 가능.
해석적 방법이 불가능 한 경우 실험적 대안.
비용이 높고 위험한 경우 사용.
다양한 환경 설정과 반복실험 가능.
장( 경제시스템 ), 단시간 시스템 연구가 가능하다.

시뮬레이션의 특징 - 단점 
개발에 많은 비용과 시간 소요.
프로그래밍 연습이라는 인상을 줄 수 있다.
과잉의 신뢰를 주는 경향이 있다.
확률적 시뮬레이션은 추정량을 산출할 뿐이다.

시뮬레이션 과정.
문제의 정의.
모델 설계.
자료 수집.
모델 변환 : 컴퓨터 처리 형태로 변환.
검증 : 컴퓨터 프로그램이 정확한가 ? ( 시뮬레이션 전용 언어를 통해 검증 )
타당성 검토 : 실시스템과 모델이 일치 ?
실험계획 수립.
실험.
결과 분석.
구현 및 문서화.

과정 틀.
실시스템 -> 모델링 - 타당성 ( validation ) -> 모델 -> 시뮬레이션 - 검증( Verification ) -> 시뮬레이터.
