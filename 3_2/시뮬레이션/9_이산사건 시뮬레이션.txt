 # 이산사건 시뮬레이션.

예시 조건.
고객 도착시간 간격 : 평균 5.0의 지수분포. ( 지수 확률 변수 )
봉사 시간 : 평균 2.5의 지수분포. ( 지수 확률 변수 )
종료 조건 : 출발사건 1,000번.

totque 계산.
고정시간 진행법 : totque = totque + quque.
가변시간 진행법( 사건중심 ) : totque = totque + queue * ( time - tlevent ) 

도착사건과 출발사건 처리의 주요변수.
tne[1] : 도착시간. ( tne[1] = time( 현재시간 ) + e_arr( 도착간격 ) )
e_arr : 도착간격. ( call expon ( seed, marrvt, e_arr ) )
e_dep : 봉사시간. ( call expon ( seed, mservt, e_dep ) )
tne[2] : 출발시간. ( tne[2] = time( 현재시간 ) + e_dep( 봉사시간 )  )

모델링 주요 변수.
numcus : 봉사를 시작한( 끝낸 ) 고객 수.
status : 창구상태.
tne[1] : 도착시간.
tno[2] : 출발시간.
rmin : 다음사건시간.
totque : 총 대기 시간.
avgwt : 평균대기시간.
avgque : 평균대기행렬길이.
marrvt : 도착시간간격.
mservt : 평균봉사시간.

사건표 작성법.
e_arr를 통해 tne[1]을 구한다.
tne[1]과 tne[2]를 비교하여 먼저 일어나는 사건을 실행한다. ( rmin = 다음사건 실행 시간 )



 # 시간 진행법의 차이.

가변시간 진행법.
종료시간 : 마지막 고객 도착. ( 효율적 )
처리 고객수 : 처음부터 알 수 있다. ( M명 )
종료시간 : 알 수 없다.
응용분야 : 몇 명까지 처리하는 경우.
특이사항 : 현재 다녀간 고객수.

고정시간 진행법.
종료시간 : 고정되어 있다. ( 많은 처리시간 소요 )
처리 고객수 : 알 수 없다.
종료시간 : 처음부터 알 수 있다. ( N분 )
응용분야 : 몇 시까지 처리하는 경우.
특이사항 : 현재 대기행렬의 크기.
