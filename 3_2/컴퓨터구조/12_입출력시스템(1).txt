
 입출력시스템.



 # 입출력시스템의 개요.

입출력시스템.
컴퓨터는 입출력 시스템을 통해 외부 데이터를 중앙처리장치나 기억장치로 입력받거나 외부로 출력.
입출력 시스템의 성능( 처리능력 )은 컴퓨터 시스템에서 매우 중요.

성능평가.
입출력 시스템의 성능평가는 입출력장치에 따라 다양.
고장 시 신속한 회복 능력, 확장성, 접근지연시간, 처리량 등.

입출력 시스템의 처리능력.
입출력 시스템의 처리능력은 장치 자체의 특성, 다른 부분과의 연결 방법,
기억장치의 계층구조, 운영체제 등에 따라 영향을 받는다.
복잡한 상업용 데이터 처리에서는 입출력 시스템의 처리 능력이 
전체 컴퓨터 시스템의 성능에 절대적인 영향을 준다.



 # 입출력시스템의 구성요소.

기본 입출력 시스템.
입출력 장치 I/O device.
입출력장치 제어기 I/O device controller.
입출력장치 인터페이스 I/O interface.
입출력 버스 I/O bux.

입출력 제어가 사용된 입출력 시스템.
입출력 장치 I/O device.
입출력장치 제어기 I/O device controller.
입출력장치 인터페이스 I/O interface.
입출력 제어기 I/O controller.
입출력 버스 I/O bux.

구성요소.
입출력 장치 : 사용자와 컴퓨터시스템의 인터페이스.
입출력장치 제어기 : 입출력장치를 구동시키는 작업을 수행하는 기기.
입출력장치 인터페이스 :  컴퓨터 내부 장치와 입출력장치의 여러 가지 차이점을 해결하기 위한 장치.
입출력 제어기 : 입출력장치 인터페이스와 컴퓨터시스템 사이에 데이터의 전송을 제어하는 장치.
입출력 버스 : 입출력장치 인터페이스와 컴퓨터시스템 사이의 데이터 전달 통로.



 # 기본 입출력 시스템.

설명.
키보드 입력장치와 모니터 출력장치를 가지고 있는 모든 컴퓨터 시스템에서 볼 수 있는 입출력 시스템.
입출력 제어기가 없기 때문에 모든 입출력 제어기능은 중앙처리장치에 두는 입출력 시스템.
중앙처리장치가 입출력을 수행하기 때문에 중앙처리장치에서 수행하는 프로그램의 수행이 지연.
키보드나 모니터와 같은 속도가 느린 입출력장치에만 사용.



 # 입출력시스템.

설명.
기본 입출력 시스템을 개선.
중앙처리장치가 입출력 수행에 개입함으로써 프로그램의 수행을 지연시키는 비효율성을 극복.
중앙처리장치가 입출력 제어기능을 수행하지 않고 별도의 입출력 제어기를 두어 입출력을 수행.



 # 입출력시스템의 구성요소의 상세설명.

입출력장치.
사용자와 컴퓨터 시스템과의 인터페이스.
중앙처리장치 또는 기억장치와 외부의 입출력 매체 사이에 정보를 전송.
중앙처리장치나 기억장치에 비해 속도가 느리고, 처리하는 데이터의 단위도 다르고, 
오류의 발생률도 높으나, 동작의 자율성을 보장할 수 있다. ( 독립성 )

입출력장치 제어기.
입출력에 필요한 입출력장치 고유의 기능만 다루는 제어기.
입출력장치의 하드웨어 기능을 제어하는 목적. 
입출력장치를 구동시키는 작업을 수행하는 기기. ( 모터를 회전, 입출력 매체를 입출력 위치에 정렬 등 )
다른 모든 장치들이 논리회로로 되어 있는 것과 달리 아날로그 회로를 포함한다.
입출력장치에 따라 입출력장치에 포함된 것과 독립적인 입출력장치 제어기가 있는 것이 있다.

입출력 제어기.
입출력장치의 공통적인 기능을 다루는 제어기.
입출력장치 인터페이스와 컴퓨터시스템 사이에 데이터의 전송을 제어하는 장치.
자체의 프로그램 수행능력을 갖고 있으며, 이는 중앙처리장치의 개입 없이 입출력이 자율적으로 행해짐.
입출력 제어기의 구현은 컴퓨터 시스템의 성능에 따라 매우 다양.
중앙처리장치, DMA 제어기, 채널 혹은 채널 제어기, 입출력 프로세서, 입출력 컴퓨터 등.



 # 입출력장치 인터페이스.

입출력장치 인터페이스.
입출력 포트( I/O port )라고도 한다.
컴퓨터 내부 장치와 외부장치와의 올바른 정보 전송을 위한 방법을 제공하는 입출력 시스템의 구성요소.
컴퓨터 내부 장치와 입출력장치의 여러 가지 차이점을 해결하기 위한 하드웨어 장치.
데이터 전송 속도 차이, 데이터 처리 단위 차이, 에러율 차이 등.

데이터 전송속도의 차이 해결 방법.
입출력장치와 중앙처리장치는 동작속도에 있어 큰 차이가 난다.
두 장치 사이에 입출력 데이터 버퍼를 두고, 이 버퍼의 상태를 나타내는 B( busy )와 D( done ) 플래그를 둔다.
B : 입출력 장치의 사용여부를 나타낸다.
D : 입출력 버퍼의 상태를 나타낸다.

데이터 처리단위의 차이 해결 방법.
주기억장치에서 데이터 단위 : 1 word.
입출력장치가 다루는 데이터 단위 : 1 byte.
입출력장치 인터페이스와 입출력 제어기 사이의 데이터 전달은 병렬로 전송한다.
입출력장치와 입출력장치 인터페이스 사이에 데이터 전달은 병렬, 직렬 둘 다 가능.

직렬 전송 serial transmission 방식.
한 비트씩 직렬로 전송.
주로 속도가 느린 키보드나 모니터와 같은 입출력장치에 사용.
데이터 전달회선의 수가 한 개이면 되기 때문에 거리가 비교적 멀어도 회선 비용이 저렴.

병렬 전송 parallel transmission 방식.
한 바이트를 한꺼번에 전송하므로 최소 8개의 데이터 전달회선이 필요.
입출력장치와 그 인터페이스의 거리가 가까운 경우에만 사용.
디스크에 사용( 거리를 짧게 하기 위해 본체에 포함 )

비트 계수기.
0부터 7까지를 셀 수 있는 카운터.
계수기에 의해 데이터 버퍼의 동작을 확인.
예를 들어 계수기가 7이면, 출력의 경우 데이터 버퍼에 있는 데이터의 출력이 완료됐음을 의미.

에러율의 차이 해결 방법.
패리티 비트( parity bit )를 이용하는 방법. 
되울림( echo back ) 방법. 
입출력 반복( retry ) 방법.
오류검사 / 수정코드 ( error detection / correction code ) 방법.

패리티 비트 방법.
입출력되는 단위 정보인 한 바이트마다 하나 이상의 비트를 추가하여 데이터 바이트와 동시에 전달하는 방법.
데이터를 송신하는 쪽에서 오류 검출을 가능하게 하는 비트를 첨부하여 송신, 수신측에서 오류의 유무를 식별.
데이터의 첨부하는 비트가 한 비트일 때 '패리비트'라고 한다.

되울림 방법.
주로 키보드 입력 장치에서 입력하는 경우에 사용.
키보드에서 입력된 문자를 CRT 화면에 나타내어 확인하는 방법.

입출력 반복 방법.
자기 디스크나 자기 테이프에서 입력할 때 사용되는 방법.
만일 한 데이터 바이트가 잘못 읽어지면 몇번이고 다시 읽어서 옳은 데이터 바이트가 읽어질 때까지 반복.
미리 정해 놓은 회수만큼 반복해도 오류가 지속될 때 타임아웃되었다고 한다.

오류검사 수정코드 방법.
패리티 비트를 사용하는 것과 유사.
데이터에 첨부해야 하는 코드의 길이가 길기 때문에 보통 입출력 데이터에 사용되지는 않는다. ( 주기억장치 )
오류검사 코드 - 데이터에 몇 개의 비트를 첨부하여 데이터에 있을 수 있는 오류를 검사.
오류수정 코드 - 오류검사뿐만 아니라 오류가 검출되었을 떄 이를 수정할 수 있는 기능까지 갖고 있다.
