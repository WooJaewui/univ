 # Shift-Reduce 구문분석.

Shift-Reduce 구문분석.
단계, 스택, 입력, 구문분석행동을 적는 구문분석표를 통해 Bottom-top 방식으로 구문분석.
reduce는 거꾸로 실행하는 것을 의미한다.

우선순위 방법.
스택의 우단에 있는 기호와 입력의 좌단에 있는 기호의 우선순위를 비교하여 shift, reduce를 실행함.
모호성을 해결할 수 있다.

순위관계 구문분석 발전단계.
연산자순위 구문분석.
단순순위 구문분석. ( 수업에서 배울 내용 )
확장순위 문법.
한정 순위 문법.
혼합 순위 문법.
LR 구문 분석.



 # 단순순위 구문분석.

단순순위 구문분석.
더 밑으로 내려 갈수록 순위가 높다.
논터미널의 값으로 같이 나오면 순위가 같다.
<, >, = 는 위에 ' . '을 찍어 일반 부등호와 구분한다.
부등호를 왼쪽, 오른쪽 기호의 위치에 맞게 사용해야된다. ( 테이블로 표현 )

단순순위 구문분석 테이블 표현.
왼쪽 기호는 row로 사용하고, 오른쪽 기호는 column으로 적어서 비교의 값을 적는다.

단순순위 구문분석 테이블 사용.
Shift-Reduce의 스택과 입력의 기호들을 비교해서 테이블 표현의 값을 확인하여
스택 기호의 우선순위가 크면 reduce, 입력 기호의 우선순위가 크면 shift한다. ( 모호성 제거 )
우선순위가 같으면 입력 기호를 shift한다.
입력에 $가 나오면 스택 기호의 reduce만 실행한다.

단순순위 구문분석 문제점.
모호성은 해결됐지만, 핸들을 결정할 수 없다. ( 잘 못 선택하면 reject 발생 )



 # LR 구문분석.

LR 구문분석.
단순순위 구무분석과 크게 다르지 않다.

SLR 파싱표를 만들기 위한 과정.
Closure.
GOTO 함수.
Canonical Col.
GOTO 그래프.

SLR 파싱표.
row : 상태를 나타낸다.
column : 기호를 나타낸다.
값 : S5, r2로 나타내고 reduce, shift를 실행하고 다음계산을 위한 상태와 핸들을 숫자를 통해 알 수 있다.
     ( 숫자만 나오는 경우도 있다 )

핸들을 알 수 있는 방법.
문법표에서 해당 상태에 어떤 기호가 핸들인지 알려준다.
